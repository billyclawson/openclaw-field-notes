## 为什么写这个？

用了 OpenClaw 三个月。从一个 Agent 扩展到五个，跑在两台 Mac 上，7×24 自动执行任务。

过程中踩过的坑，比写出来的代码还多。

网上关于 OpenClaw 的内容，要么是官方文档（告诉你 **能** 做什么），要么是推文截图（告诉你别人 **做了** 什么）。几乎没人写：**实际操作中会遇到什么问题，以及怎么解决。**

这本手册就是填这个空白的。

## 环境概览

- **硬件：** MacBook Air M2 (调度中心) + Mac Mini M2 Pro (工作节点)
- **Agent 数量：** 5 — Billy (调度), Gus (工程), Jason (研究), Dora (营销), Hugo (QA)
- **通信：** Discord (Agent 间) + Telegram (人机接口)
- **模型：** Kimi 2.5 为主，OpenAI Codex 备选，MiniMax 兜底
- **月成本：** ~$30-50（从最高 $200+ 降下来的）
- **适用版本：** OpenClaw 2026.3.x（测试于 2026.3.2）

> 所有内容基于真实运行数据，不是 demo，不是理论。

## 社区资源

找不到答案？试试这些地方：

- **OpenClaw 官方文档：** [docs.openclaw.ai](https://docs.openclaw.ai) — 最权威的功能说明

- **GitHub Discussions：** [github.com/anthropics/claude-code/discussions](https://github.com/anthropics/claude-code/discussions) — 社区讨论、最佳实践
- **Discord 社区：** OpenClaw 官方 Discord — 实时交流、问题求助

### 中文社区

国内用户经常遇到网络、API 代理等特殊问题，以下中文资源可能更有针对性：

- **V2EX：** 搜索 "OpenClaw" 或 "Claude Code" — 中文开发者讨论
- **掘金 / CSDN：** 有用户写过安装和配置教程，适合入门
- **飞书知识库：** 部分团队在飞书上分享了 Multi-Agent 实践文档
- **即刻 / 小红书：** 搜索 "AI Agent 自动化" — 有不少实战分享

### 国内消息平台集成

OpenClaw 的通信渠道不只有 Discord 和 Telegram。国内平台正在逐步支持 Agent 接入：

| 平台 | 状态 | 说明 |
|------|------|------|
| **飞书 (Lark)** | ✅ 可用 | 原生支持 Bot，适合团队协作场景 |
| **企业微信 (WeCom)** | ✅ 可用 | 企业版支持 Bot API，自建应用即可接入 |
| **钉钉 (DingTalk)** | ✅ 可用 | 支持自定义机器人和 Webhook |
| **QQ** | 🟡 测试中 | QQ 官方机器人平台已开放，部分功能可用 |
| **微信** | 🟡 内测中 | 微信正在内部测试 AI Bot 直接接入的能力 |

> **推荐：** 如果你在国内团队中使用，**飞书** 是目前最成熟的选择 — Bot API 完善，消息格式丰富，支持卡片消息和交互式组件。
